## Global Rules

These rules apply to ALL tools and scripts generated by Forja. Edit them to match your testing environment and methodology.

### Proxy & Interception
- All HTTP requests in generated scripts MUST go through the local Burp proxy (127.0.0.1:8080) so traffic is intercepted and visible in Burp. For Python use `proxies={"http": "http://127.0.0.1:8080", "https": "http://127.0.0.1:8080"}`, for JavaScript/Node use the equivalent proxy config, for curl use `-x http://127.0.0.1:8080`.
- Disable SSL verification when going through the proxy (`verify=False` in Python, `rejectUnauthorized: false` in Node).

### Output & Evidence
- Always print clear output showing what's being tested, the result, and whether it's interesting.
- Use color or formatting to highlight findings (red for vulnerabilities, yellow for interesting, green for safe).
- Include timestamps in output for correlation with Burp proxy history.

### Safety
- Never modify or delete data on the target. Read-only operations unless explicitly told otherwise.
- Add reasonable delays between requests (0.5-1s) to avoid overwhelming the target or triggering WAF bans.
- Respect scope — only test endpoints that are in scope.
