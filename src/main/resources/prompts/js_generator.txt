Replace TOOL_NAME with the actual tool name (e.g., "Traffic Sniffer", "SQLi PoC", "JWT Analyzer").

## Logging API

Use these functions throughout the script to report to the panel:

- `forjaLog(message, type)` — Log a message. Types: 'info', 'success', 'warn', 'error', 'critical', 'data', 'header'
- `forjaStatus(text)` — Update the status bar text
- `forjaKV(key, value, type)` — Log a key-value pair (great for displaying request details, findings, parameters)

## Script requirements

1. Use modern JavaScript (ES6+) with fetch API for HTTP requests
2. Be self-contained and runnable in a browser console or injected via Burp proxy
3. Include clear comments explaining what each section does
4. Handle errors gracefully — catch and log errors to the panel, never crash silently
5. Include configurable variables at the top (target URL, tokens, wordlists, etc.)
6. Output results to BOTH the floating panel AND the browser console
7. Keep under 200 lines. Focus on core functionality.
8. For tools that make requests: add configurable delay between requests (default 500ms) to avoid overwhelming the target
9. For sniffers/interceptors: show method, URL, status code, content-type, interesting headers, body size. Highlight sensitive data patterns (tokens, passwords, emails, SSNs, credit cards).
10. For PoCs: show step-by-step attack flow, highlight each vulnerable parameter, show evidence of exploitation.
11. When creating action buttons in the toolbar, use the Burp orange (#e8a238) for primary actions.

## Script types

- **traffic-sniffer**: Intercept and log XHR/fetch requests and responses using monkey-patching. Show method, URL, status, interesting headers, body preview. Flag sensitive data. Use forjaKV for structured display.
- **poc-exploit**: Proof-of-concept demonstrating a specific vulnerability. Show attack steps, payloads, and evidence in the panel. Include toolbar buttons for Start/Stop.
- **jwt-manipulator**: Decode, display, and test JWT tokens from cookies/localStorage/headers. Show decoded header, payload, expiry, algorithm. Flag weak algorithms (none, HS256 with short key). Include toolbar buttons to test token modifications.
- **request-replayer**: Replay and modify captured requests. Show original vs modified responses. Include controls for parameter manipulation.
- **parameter-fuzzer**: Fuzz parameters with payloads (SQLi, XSS, SSTI). Show each test case result. Flag interesting responses (different status codes, error messages, reflection).
